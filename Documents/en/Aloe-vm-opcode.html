AloeVM Instruction Set (Complete Table v0.1)

  • The instruction ID is 2 bytes long. The first byte is the category, and the second byte is the index within that category (the order in the opcode table).
  • Any category + index combination that is not listed in this file is considered unused / reserved in the current version.
  • For multi-byte immediate operands, values are stored in little-endian format, consistent with the main AloeVM specification.
  • The semantics of each instruction and the overall VM behavior are governed by the main AloeVM specification (especially Chapters 1–7).

Category 0x01: Stack / Const

Category Index Mnemonic Operand Description
0x01 0x00 PUSH_I32_0 None Pushes the int32 value 0 onto the operand stack.
0x01 0x01 PUSH_I32_1 None Pushes the int32 value 1 onto the operand stack.
0x01 0x02 PUSH_I32_M1 None Pushes the int32 value -1 onto the operand stack.
0x01 0x03 PUSH_I32 int32 immediate (4 bytes) Pushes the specified int32 immediate value onto the operand stack.
0x01 0x04 PUSH_F32 float32 immediate (4 bytes) Pushes the specified float32 immediate value onto the operand stack.
0x01 0x05 PUSH_DECIMAL decimal immediate (16 bytes, implementation dependent) Pushes the specified decimal immediate value onto the operand stack.
0x01 0x06 PUSH_BOOL_FALSE None Pushes the bool value false onto the operand stack.
0x01 0x07 PUSH_BOOL_TRUE None Pushes the bool value true onto the operand stack.
0x01 0x08 PUSH_NULL None Pushes a null reference onto the operand stack (for reference types only).
0x01 0x09 PUSH_CONST Constant pool index (uint16) Loads the value at the specified constant pool index and pushes it onto the operand stack.
0x01 0x0A DUP None Duplicates the top value on the operand stack.
0x01 0x0B POP None Pops and discards the top value of the operand stack.
0x01 0x0C SWAP None Swaps the order of the top two values on the operand stack.

Category 0x02: Load / Store (locals, arguments, fields)

Category Index Mnemonic Operand Description
0x02 0x00 LD_LOCAL Local index (uint16) Loads the value of the specified local variable and pushes it onto the operand stack.
0x02 0x01 ST_LOCAL Local index (uint16) Stores the top of the operand stack into the specified local variable and pops it from the stack.
0x02 0x02 LD_ARG Argument index (uint16) Loads the value of the specified argument and pushes it onto the operand stack.
0x02 0x03 ST_ARG Argument index (uint16) Stores the top of the operand stack into the specified argument slot (used in special cases).
0x02 0x04 LD_FIELD Field ID (uint16) Loads the specified field from the object reference at the top of the stack and pushes the field value.
0x02 0x05 ST_FIELD Field ID (uint16) Stores the top stack value into the specified field of the object reference just below it, then pops the value.
0x02 0x06 LD_STATIC_FIELD Field ID (uint16) Loads the value of the specified static field and pushes it onto the operand stack.
0x02 0x07 ST_STATIC_FIELD Field ID (uint16) Stores the top stack value into the specified static field and pops it from the stack.
0x02 0x08 LD_THIS None Pushes the current instance method’s this reference onto the operand stack.
0x02 0x09 LD_ELEM None Pops an index and an array reference from the stack, then pushes the element value at that index.
0x02 0x0A ST_ELEM None Pops a value, an index, and an array reference, then stores the value into the array element at that index.
0x02 0x0B LD_LOCAL_0 None Loads local variable 0 and pushes it onto the stack (short form).
0x02 0x0C LD_LOCAL_1 None Loads local variable 1 and pushes it onto the stack (short form).
0x02 0x0D ST_LOCAL_0 None Stores the top stack value into local variable 0 and pops it (short form).
0x02 0x0E ST_LOCAL_1 None Stores the top stack value into local variable 1 and pops it (short form).

Category 0x03: Arithmetic / Bit

Category Index Mnemonic Operand Description
0x03 0x00 ADD_I32 None Pops two int32 values and pushes their sum.
0x03 0x01 SUB_I32 None Pops two int32 values and pushes (second − first).
0x03 0x02 MUL_I32 None Pops two int32 values and pushes their product.
0x03 0x03 DIV_I32 None Pops two int32 values and pushes (second ÷ first). Division by zero raises an exception.
0x03 0x04 MOD_I32 None Pops two int32 values and pushes (second % first).
0x03 0x05 NEG_I32 None Negates the top int32 on the stack and pushes the result.
0x03 0x06 ADD_F32 None Pops two float32 values and pushes their sum.
0x03 0x07 SUB_F32 None Pops two float32 values and pushes (second − first).
0x03 0x08 MUL_F32 None Pops two float32 values and pushes their product.
0x03 0x09 DIV_F32 None Pops two float32 values and pushes (second ÷ first).
0x03 0x0A NEG_F32 None Negates the top float32 on the stack and pushes the result.
0x03 0x0B ADD_DECIMAL None Pops two decimal values and pushes their sum.
0x03 0x0C SUB_DECIMAL None Pops two decimal values and pushes (second − first).
0x03 0x0D MUL_DECIMAL None Pops two decimal values and pushes their product.
0x03 0x0E DIV_DECIMAL None Pops two decimal values and pushes (second ÷ first).
0x03 0x0F AND_I32 None Pops two int32 values and pushes their bitwise AND.
0x03 0x10 OR_I32 None Pops two int32 values and pushes their bitwise OR.
0x03 0x11 XOR_I32 None Pops two int32 values and pushes their bitwise XOR.
0x03 0x12 NOT_I32 None Applies bitwise NOT to the top int32 on the stack and pushes the result.
0x03 0x13 SHL_I32 None Pops a shift amount and an int32 value, then pushes (second << first).
0x03 0x14 SHR_I32 None Pops a shift amount and an int32 value, then pushes (second >> first).

Category 0x04: Compare

Category Index Mnemonic Operand Description
0x04 0x00 CMP_EQ None Pops two values and pushes bool true if they are equal, otherwise false.
0x04 0x01 CMP_NE None Pops two values and pushes bool true if they are not equal.
0x04 0x02 CMP_LT None Pops two values and pushes bool true if (second < first).
0x04 0x03 CMP_LE None Pops two values and pushes bool true if (second ≤ first).
0x04 0x04 CMP_GT None Pops two values and pushes bool true if (second > first).
0x04 0x05 CMP_GE None Pops two values and pushes bool true if (second ≥ first).
0x04 0x06 CMP_NULL None Pushes bool true if the top reference value is null, otherwise false.
0x04 0x07 CMP_NOT_NULL None Pushes bool true if the top reference value is not null.

Category 0x05: Control Flow

Category Index Mnemonic Operand Description
0x05 0x00 JMP Relative offset (int32) Unconditional branch. Adds the specified offset to the instruction pointer.
0x05 0x01 JMP_IF_TRUE Relative offset (int32) Pops a bool from the stack. If it is true, branches by the specified offset.
0x05 0x02 JMP_IF_FALSE Relative offset (int32) Pops a bool from the stack. If it is false, branches by the specified offset.
0x05 0x03 JMP_SWITCH Table index / branch table (implementation dependent) Uses an integer value from the stack as an index into a branch table and jumps to the corresponding label.
0x05 0x04 JMP_IF_ZERO Relative offset (int32) Pops an integer value and branches if the value is 0.
0x05 0x05 JMP_IF_NONZERO Relative offset (int32) Pops an integer value and branches if the value is not 0.

Category 0x06: Call / Return

Category Index Mnemonic Operand Description
0x06 0x00 CALL Function ID (uint16) Calls the specified function ID and creates a new frame.
0x06 0x01 CALL_VIRT Slot ID (uint16) Resolves and dispatches a virtual method call based on the this reference on the stack.
0x06 0x02 CALL_INTERFACE Slot ID (uint16) Performs dynamic dispatch via an interface method slot.
0x06 0x03 CALL_INDIRECT Signature ID (uint16) Uses a function handle from the stack to perform an indirect function call.
0x06 0x04 RET None Returns from the current frame, leaving exactly one return value on the caller’s stack.
0x06 0x05 RET_VOID None Returns from the current frame without a return value.
0x06 0x06 TAILCALL Function ID (uint16) Performs a tail call to the specified function, reusing the current frame (optimization hint).

Category 0x07: Object / Array

Category Index Mnemonic Operand Description
0x07 0x00 NEWOBJ Type ID (uint16) Allocates an instance of the specified class type on the heap and pushes its reference.
0x07 0x01 NEWSTRUCT Type ID (uint16) Creates an instance of the specified struct type, either on the heap or the stack (implementation dependent).
0x07 0x02 NEW_ARRAY Element type ID (uint16) Pops a length from the stack, allocates an array of that length, and pushes the array reference.
0x07 0x03 NEW_ZEROARRAY Element type ID (uint16) Allocates an array of the given length and zero-initializes all elements (same as NEW_ARRAY but emphasizes zeroing).
0x07 0x04 BOX Type ID (uint16) Boxes the top value-type on the stack into a reference type allocated on the heap.
0x07 0x05 UNBOX Type ID (uint16) Unboxes the top reference on the stack to the specified value-type and pushes the value-type.
0x07 0x06 CAST_CLASS Type ID (uint16) Casts the top reference on the stack to the specified class type; throws if the cast fails.
0x07 0x07 IS_INSTANCE Type ID (uint16) Pushes bool true if the top reference is an instance of the specified type, otherwise false.
0x07 0x08 CLONE_OBJ None Clones the object reference at the top of the stack, pushing the new reference (deep or shallow copy is implementation dependent).

Category 0x08: Pipe I/O (basic pipe input/output)

Corresponds to the “Pipe / Filter” representation and “5.4 Pipe lifecycle and VM instructions” sections in the main AloeVM specification.
Lifecycle-related instructions (create / connect / disconnect / delete) are grouped under Category 0x09.
Category Index Mnemonic Operand Description
0x08 0x00 PIPE_READ None Pops a pipe<T> from the stack, reads one element, and pushes the element value (blocking).
0x08 0x01 PIPE_WRITE None Pops a value and a pipe<T> from the stack, writes the value into the pipe, and may block if necessary.
0x08 0x02 PIPE_TRY_READ None Attempts a non-blocking read from a pipe. On success, pushes (value, bool true); on failure, pushes only bool false.
0x08 0x03 PIPE_TRY_WRITE None Attempts a non-blocking write to a pipe and pushes a bool indicating success.
0x08 0x04 PIPE_CLOSE_READ None Closes the read side of the pipe reference at the top of the stack.
0x08 0x05 PIPE_CLOSE_WRITE None Closes the write side of the pipe reference at the top of the stack.
0x08 0x06 PIPE_GET_STATE None Pushes an integer or enum describing the current state of the pipe (open / closed / error, etc.).

Category 0x09: Pipe / Filter / Producer / Consumer (lifecycle, pipeline construction)

Corresponds to “5.4 Pipe lifecycle and VM instructions” in the main AloeVM specification.
Covers pipe creation, connection, disconnection, deletion, and starting/stopping filter / producer / consumer tasks.
Category Index Mnemonic Operand Description
0x09 0x00 PIPE_NEW Element type ID (uint16) Creates a new pipe<T> and pushes its reference (lifecycle: “create”).
0x09 0x01 PIPE_FROM_HANDLE Host handle ID (uint16 or similar, implementation dependent) Creates an AloeVM pipe value from a host-side pipe resource.
0x09 0x02 PIPE_CONNECT None Takes two pipe references from the stack and connects them as upstream → downstream (lifecycle: “connect”).
0x09 0x03 PIPE_DISCONNECT None Takes two pipe references from the stack and disconnects them (lifecycle: “disconnect”).
0x09 0x04 PIPE_DELETE None Marks the pipe at the top of the stack as logically removed from the pipeline and a candidate for resource release (lifecycle: “delete”).
0x09 0x05 FILTER_NEW Filter definition ID (uint16) Creates a filter instance from the specified filter definition and pushes it onto the stack.
0x09 0x06 FILTER_BIND None Pops an input pipe, an output pipe, and a filter from the stack, then binds the pipes to the filter.
0x09 0x07 FILTER_START None Starts executing the filter at the top of the stack as a new task or thread.
0x09 0x08 FILTER_STOP None Stops the running filter (intended as a graceful cancellation).
0x09 0x09 PRODUCER_START None Starts the producer at the top of the stack and begins writing to its associated pipe(s).
0x09 0x0A PRODUCER_STOP None Stops the producer and, if appropriate, closes the write side of its pipe(s).
0x09 0x0B CONSUMER_START None Starts the consumer at the top of the stack and begins reading from its pipe(s).
0x09 0x0C CONSUMER_STOP None Stops the consumer and, if appropriate, closes the read side of its pipe(s).
0x09 0x0D PIPE_AWAIT_COMPLETION None Waits for the completion of the pipe or the entire pipeline at the top of the stack (synchronization point).
0x09 0x0E PIPE_CANCEL None Sends a cancellation request to the pipe or pipeline at the top of the stack.
0x09 0x0F PIPE_FLUSH None Flushes the buffer of the pipe at the top of the stack, pushing as much data downstream as possible.

Category 0x0A: Exception

Category Index Mnemonic Operand Description
0x0A 0x00 THROW None Throws the exception object at the top of the stack.
0x0A 0x01 THROW_NEW Exception type ID (uint16) Creates an instance of the specified exception type and throws it immediately (message etc. are implementation dependent).
0x0A 0x02 RETHROW None Re-throws the currently handled exception from within a catch block.
0x0A 0x03 LEAVE Relative offset (int32) Leaves a try/finally block and, after executing the finally block, transfers control to the specified offset.

Category 0x0F: Misc / System (miscellaneous, debugging, host interop)

Category Index Mnemonic Operand Description
0x0F 0x00 NOP None Does nothing. Used for alignment or leaving patchable gaps.
0x0F 0x01 BREAK None If a debugger is attached, signals a breakpoint (implementation dependent).
0x0F 0x02 DEBUG_TRACE Trace ID (uint16) Calls an implementation-dependent trace hook. Useful for logging and diagnostics.
0x0F 0x03 HALT Exit code (int32) Immediately stops VM execution and returns the exit code to the host.
0x0F 0x04 ENTER_UNMANAGED Channel ID etc. (implementation dependent) Performs pre-processing before entering a host-native code region. Used for WASM or FFI session start.
0x0F 0x05 LEAVE_UNMANAGED None Used as a pair with ENTER_UNMANAGED to end a native session.
