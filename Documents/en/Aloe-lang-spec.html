Aloe Standard Library Specification (Filters Edition)

Draft • English • Filters Library • version 3
This document is a “Standard Filter Library Specification” that is separate from the Aloe language specification.

0. Overview

This document defines the standard filter set of the Aloe language, assuming the existence of pipe<T> and filter.

The core language specification defines syntax, the type system, and pipe / filter syntax, while this document independently specifies the names, behavior, and types of the filters that are provided as the standard library.

Implementations should aim to be “as faithful as possible” to the signatures and semantics defined here, but some behavior may be implementation-dependent due to performance or platform differences.

NOTE: All code examples in this document are Aloe pseudo code, and may be updated in the future as the language specification evolves.

1. Common Rules for Filters

1.1 Call Syntax

All filters in this document are applied to pipe<T> using the form filter(NAME ...). 

// Full form
stdin
    | filter(utf8)
    | filter(lineSplit)
    | filter(trim)
    | stdout;

However, inside a pipeline statement, the keyword filter may be omitted, and a shorthand syntax |: NAME ... can be used. 

// Shorthand (pipeline-only syntax)
stdin
    |: utf8
    |: lineSplit
    |: trim
    | stdout;

The following two snippets are equivalent: 

stdin
    | filter(utf8)
    | filter(json: User);

stdin
    |: utf8
    |: json: User;

Filters that require a type argument receive it as a labeled argument: 

stdin
    |: utf8
    |: json: User
    | users;

1.2 Type Notation

This document uses a simplified notation for types. Examples:

  • pipe<string> – a stream of strings
  • pipe<byte> – a stream of bytes
  • pipe<T> – a pipe with a type parameter

1.3 Side Effects and Exceptions

Filters may behave in several ways, such as:

  • Consuming the entire input pipe before producing any output (e.g. sort, shuffle, zip)
  • Letting only a subset of elements pass through (e.g. if, startswith, contains)
  • Holding internal state (e.g. AESencrypt with an encryption context)

How filters handle errors (exceptions, returning Result<T>, etc.) is mentioned in each filter’s description when needed.

1.4 Namespace

The root namespace of the standard library is aloe. Every standard filter belongs to some namespace under this root.

  • Namespace names are all lowercase, separated by ..
  • Examples: aloe.text, aloe.encoding, aloe.crypto.hash, etc.

For each filter, this document assigns a canonical fully-qualified name of the form aloe.xxx.yyy. How imports and short names (using only the simple filter name) work in actual code is defined on the language-spec side.

2. Text / Encoding

2.1 utf8

Namespace / Signature

Namespace: aloe.encoding
filter utf8 : pipe<byte> -> pipe<string>

Decodes the input byte stream as UTF-8 and converts it into a stream of strings. 

stdin
    |: utf8
    |: lineSplit
    | stdout;

2.2 lineSplit

Namespace / Signature

Namespace: aloe.text
filter lineSplit : pipe<string> -> pipe<string>

Splits string chunks on line breaks and outputs one line of text per element. How line breaks (\n vs \r\n) are treated is implementation-dependent, but at minimum \n must be recognized. 

stdin
    |: utf8
    |: lineSplit
    |: trim
    | stdout;

3. JSON / Data Transformation

3.1 json (decode)

Namespace / Signature

Namespace: aloe.json
filter json<T> : pipe<string> -> pipe<T>

Decodes JSON text into typed objects. At call sites, the target type is passed as in filter(json: User). 

stdin
    |: utf8
    |: json: User
    | users;    // pipe<User>

3.2 CSV: csvenode / csvdecode

3.2.1 csvenode

Namespace: aloe.csv
filter csvenode : pipe<string[]> -> pipe<string>

Encodes a string array into a single CSV line. Delimiters and escape rules are implementation-dependent, but following a standard CSV specification is recommended.

3.2.2 csvdecode

Namespace: aloe.csv
filter csvdecode : pipe<string> -> pipe<string[]>

Parses each input line as CSV and outputs an array of column strings. 

stdin
    |: utf8
    |: lineSplit
    |: csvdecode
    | rows;   // pipe<string[]>

4. Conditionals / Reordering

4.1 if / elseif / else

These filters form a composite “conditional pass-through” construct: elements are passed through depending on conditions.

Namespace / Signature (conceptual)

Namespace: aloe.filter
filter if : pipe<T> -> pipe<T>
filter elseif : pipe<T> -> pipe<T>
filter else : pipe<T> -> pipe<T> 

users
    |: if:     .Age >= 20 and .Gender == Gender.Male
    |: elseif: .Age >= 13
    |: else
    | stdout;
  • if – first conditional branch.
  • elseif – additional conditional branch, valid only if it directly follows an if or another elseif.
  • else – passes elements that did not match any prior branch.

4.2 switch

Namespace: aloe.filter
filter switch : pipe<T> -> pipe<T> (detailed spec to be added later)

switch is a filter that extracts a key from each element and dispatches it to one of multiple branches based on that key. At the moment, only the name and rough role are reserved; details such as the syntax of case clauses are to be defined in the language specification later.

4.3 sort

Namespace / Signature

Namespace: aloe.collection
filter sort : pipe<T> -> pipe<T>
filter sort(key) : pipe<T> -> pipe<T>

Sorts input elements and outputs them in order. Internally, all elements are buffered, sorted, then emitted. 

users
    |: sort: .Age   // sort ascending by Age
    | stdout;

4.4 shuffle

Namespace: aloe.collection
filter shuffle : pipe<T> -> pipe<T>

Randomly shuffles the order of elements. Internally buffers all elements, randomizes their order, then outputs them. 

cards
    |: shuffle
    | gameDeck;

5. Crypto / Hash / Binary

5.1 Hash: md4 / sha1 / sha256

Namespace: aloe.crypto.hash
filter md4 : pipe<byte> -> pipe<byte>
filter sha1 : pipe<byte> -> pipe<byte>
filter sha256 : pipe<byte> -> pipe<byte>

Computes a hash over the entire input and outputs the resulting hash bytes. For security purposes, sha256 is recommended; md4 and sha1 are provided mainly for compatibility. 

stdin
    |: sha256
    |: bin2hex
    | stdout;

5.2 Byte-Stream Conversions: bin2hex / crc32

5.2.1 bin2hex

Namespace: aloe.encoding
filter bin2hex : pipe<byte> -> pipe<string>

Converts bytes to their hexadecimal string representation.

5.2.2 crc32

Namespace: aloe.crypto.hash
filter crc32 : pipe<byte> -> pipe<uint32>

Calculates the CRC32 value over the entire input and outputs it as a single integer.

5.3 Base64 / URL

Namespace: aloe.encoding
filter base64encode : pipe<byte> -> pipe<string>
filter base64decode : pipe<string> -> pipe<byte>

Namespace: aloe.net.http
filter urlencode : pipe<string> -> pipe<string>
filter urldecode : pipe<string> -> pipe<string> 

fileBytes
    |: base64encode
    | stdout;

// ...

stdin
    |: base64decode
    | fileBytes;

5.4 Symmetric Crypto: Blowfish / AES

5.4.1 Blowfish: blowfishencrypt / blowfishdecrypt

Namespace: aloe.crypto.cipher
filter blowfishencrypt(ctx: BlowfishContext) : pipe<byte> -> pipe<byte>
filter blowfishdecrypt(ctx: BlowfishContext) : pipe<byte> -> pipe<byte>

Uses ctx (holding key, IV, mode, etc.) to encrypt or decrypt data.

5.4.2 AES: AESencrypt / AESdecrypt

Namespace: aloe.crypto.cipher
filter AESencrypt(ctx: AESContext) : pipe<byte> -> pipe<byte>
filter AESdecrypt(ctx: AESContext) : pipe<byte> -> pipe<byte>

Encrypts/decrypts using AES (CBC / GCM, etc.). The specific mode and padding are defined by AESContext.

6. String Utilities

6.1 Case Conversion: tolower / toupper

Namespace: aloe.text
filter tolower : pipe<string> -> pipe<string>
filter toupper : pipe<string> -> pipe<string> 

lines
    |: tolower
    | normalized;

6.2 Case Styles: camelcase / pascalcase / snakecase

Namespace: aloe.text
filter camelcase : pipe<string> -> pipe<string>
filter pascalcase : pipe<string> -> pipe<string>
filter snakecase : pipe<string> -> pipe<string>

  • camelcase: "hello world" → "helloWorld"
  • pascalcase: "hello world" → "HelloWorld"
  • snakecase: "Hello World" → "hello_world"

6.3 Substring/Match Filters: startswith / contains / strstr

Namespace: aloe.text
filter startswith(prefix: string) : pipe<string> -> pipe<string>
filter contains(needle: string) : pipe<string> -> pipe<string>
filter strstr(needle: string) : pipe<string> -> pipe<string>

  • startswith: passes only lines that start with prefix.
  • contains: passes only lines that contain needle.
  • strstr: returns the substring starting from the position where needle first appears (similar to PHP’s strstr).
lines
    |: startswith: "[ERROR]"
    |: strstr: "detail="
    | stdout;

6.4 Trim / Join: ltrim / rtrim / trim / join

Namespace: aloe.text
filter ltrim : pipe<string> -> pipe<string>
filter rtrim : pipe<string> -> pipe<string>
filter trim : pipe<string> -> pipe<string>
filter join(sep: string) : pipe<string> -> pipe<string>

join concatenates all input strings with sep and outputs a single string. 

lines
    |: trim
    |: join: "\n"
    | text;

7. Archive / Compression / CSV

7.1 ZIP / TAR: zip / unzip / tar

Namespace: aloe.archive
filter zip : pipe<byte> -> pipe<byte>
filter unzip : pipe<byte> -> pipe<byte>
filter tar : pipe<byte> -> pipe<byte>

Here we assume a simple model in which “the entire stream” is treated as a single archive. More advanced operations (multiple entries, file names, etc.) may be provided as separate high-level APIs. 

stdin
    |: zip
    | stdout;

// ...

stdin
    |: unzip
    | stdout;

7.2 CSV: csvenode / csvdecode

(Described in section 3.2; only re-listed here as part of this category.)

8. HTML / Sanitization

8.1 Escape / Unescape

Namespace: aloe.html
filter htmlspecialchars : pipe<string> -> pipe<string>
filter htmlspecialchars_decode : pipe<string> -> pipe<string>
filter html_entity_decode : pipe<string> -> pipe<string>
filter strip_tags : pipe<string> -> pipe<string>

  • htmlspecialchars: escapes characters like <, >, &, " into safe entities.
  • htmlspecialchars_decode: reverses htmlspecialchars.
  • html_entity_decode: decodes a broader set of HTML entities (e.g. &nbsp;).
  • strip_tags: strips HTML tags and leaves only plain text.
htmlLines
    |: strip_tags
    |: trim
    | plainText;

9. Parsing / Formatting

9.1 Query String: parse_str

Namespace: aloe.net.http
filter parse_str : pipe<string> -> pipe<map<string, string>>

Parses a query string like "foo=1&bar=2" into a map of keys and values.

9.2 Number Formatting: number_format

Namespace: aloe.number
filter number_format(spec: string) : pipe<decimal> -> pipe<string>

Formats numbers into strings. The format string spec is implementation-dependent, but something like "#,##0.00" can be assumed as an example.

9.3 Regex Escaping: quote_meta

Namespace: aloe.text
filter quote_meta : pipe<string> -> pipe<string>

Escapes characters that are special in the regex engine, so they can be safely treated as literals. 

patterns
    |: quote_meta
    | safePatterns;

Filters not listed in this document may be freely added as user-defined filters or by external libraries. The names defined here are reserved as “Aloe standard filters” and are intended to act as a common API surface across compatible implementations.
