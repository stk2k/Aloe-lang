Aloe Standard Library Specification (Filters)

Draft • English • Filters Library • version 3
This document is a separate “standard filters library specification” independent from the Aloe language specification.

0. Overview

This document defines a set of standard filters in Aloe, based on the concepts of pipe<T> and filter.

The core language specification defines syntax, the type system, and the pipe / filter constructs. This document is independent of that and describes the names, behavior, and types of the filters that are provided by default.

Implementations should aim to be “as faithful as possible” to the signatures and semantics defined here, but some behavior may depend on the implementation due to performance or platform differences.

NOTE: All code examples in this document are Aloe pseudo code and may be updated in the future as the language specification evolves.

1. Common Rules for Filters

1.1 Invocation syntax

All filters in this document are applied to a pipe<T> using the filter(NAME ...) form. 

// Full form
stdin
    | filter(utf8)
    | filter(lineSplit)
    | filter(trim)
    | stdout;

However, within a pipeline statement the keyword filter may be omitted, and the sugar syntax |: NAME ... can be used instead. 

// Short form (pipeline-only sugar)
stdin
    |: utf8
    |: lineSplit
    |: trim
    | stdout;

The following two snippets are equivalent: 

stdin
    | filter(utf8)
    | filter(json: User);

stdin
    |: utf8
    |: json: User;

Filters that require type parameters receive the type as a labeled argument: 

stdin
    |: utf8
    |: json: User
    | users;

1.2 Type notation

For simplicity, types are written in a compact form in this document. Examples:

  • pipe<string> – a pipe of strings
  • pipe<byte> – a pipe of bytes
  • pipe<T> – a pipe parameterized by type T

1.3 Side effects and exceptions

Filters may exhibit the following behaviors:

  • Consume the entire input pipe before producing output (e.g. sort, shuffle, zip)
  • Pass through only a subset of the input (e.g. if, startswith, contains)
  • Maintain internal state (e.g. ciphers with contexts like AESencrypt)

How errors are handled (exceptions vs returning Result<T>, etc.) may be described individually for each filter when relevant.

1.4 Namespaces

The root namespace of the standard library is aloe. Every standard filter belongs to a namespace under this root.

  • Namespace names are all lowercase, and segments are separated by ..
  • Examples: aloe.text, aloe.encoding, aloe.crypto.hash, etc.

This document assigns each filter a “canonical fully-qualified name” of the form aloe.xxx.yyy. How imports work or how these names can be shortened in actual code is defined in the language specification.

2. Text / Encoding

2.1 utf8

Namespace / Signature

Namespace: aloe.encoding
filter utf8 : pipe<byte> -> pipe<string>

Decodes the input byte stream as UTF-8 and produces a stream of strings.

stdin
    |: utf8
    |: lineSplit
    | stdout;

2.2 lineSplit

Namespace / Signature

Namespace: aloe.text
filter lineSplit : pipe<string> -> pipe<string>

Splits string chunks into individual lines and emits one line per element. The handling of line endings (\n / \r\n) is implementation-dependent, but at minimum \n must be treated as a separator. 

stdin
    |: utf8
    |: lineSplit
    |: trim
    | stdout;

3. JSON / Data Transformation

3.1 json (decode)

Namespace / Signature

Namespace: aloe.json
filter json<T> : pipe<string> -> pipe<T>

Decodes JSON text into typed objects. At the call site, the type is passed as in filter(json: User). 

stdin
    |: utf8
    |: json: User
    | users;    // pipe<User>

3.2 CSV: csvenode / csvdecode

3.2.1 csvenode

Namespace: aloe.csv
filter csvenode : pipe<string[]> -> pipe<string>

Encodes a string array into a single CSV line. Separators and escaping rules are implementation-dependent, but should follow a standard CSV specification.

3.2.2 csvdecode

Namespace: aloe.csv
filter csvdecode : pipe<string> -> pipe<string[]>

Parses each line as CSV and emits an array of columns. 

stdin
    |: utf8
    |: lineSplit
    |: csvdecode
    | rows;   // pipe<string[]>

4. Conditional / Ordering

4.1 if / elseif / else

These filters form a group that controls which elements pass through based on conditions. They are interpreted as a single composite filter.

Namespace / Signatures (conceptual)

Namespace: aloe.filter
filter if : pipe<T> -> pipe<T>
filter elseif : pipe<T> -> pipe<T>
filter else : pipe<T> -> pipe<T> 

users
    |: if:     .Age >= 20 and .Gender == Gender.Male
    |: elseif: .Age >= 13
    |: else
    | stdout;
  • if is the first conditional branch.
  • elseif is an additional branch that is valid only when it directly follows an if or another elseif.
  • else passes elements that did not match any previous condition.

4.2 switch

Namespace: aloe.filter
filter switch : pipe<T> -> pipe<T> (detailed specification to be added)

switch retrieves a key from each element and routes it to one of multiple branches based on that key. For now, only the name and rough role are reserved; the exact syntax (how to write case clauses, etc.) will be defined in the language specification.

4.3 sort

Namespace / Signature

Namespace: aloe.collection
filter sort : pipe<T> -> pipe<T>
filter sort(key) : pipe<T> -> pipe<T>

Sorts the input elements and emits them in sorted order. Internally, the filter buffers all elements, sorts them, and then outputs them. 

users
    |: sort: .Age   // sort by Age ascending
    | stdout;

4.4 shuffle

Namespace: aloe.collection
filter shuffle : pipe<T> -> pipe<T>

Randomly permutes the order of elements. Internally, it buffers all elements and then outputs them in randomized order. 

cards
    |: shuffle
    | gameDeck;

5. Crypto / Hash / Binary

5.1 Hash: md4 / sha1 / sha256

Namespace: aloe.crypto.hash
filter md4 : pipe<byte> -> pipe<byte>
filter sha1 : pipe<byte> -> pipe<byte>
filter sha256 : pipe<byte> -> pipe<byte>

Computes a hash value for the entire input and outputs the hash as a byte stream. For security purposes, sha256 is recommended, while md4 and sha1 should be treated as legacy/compatibility options. 

stdin
    |: sha256
    |: bin2hex
    | stdout;

5.2 Byte-level conversion: bin2hex / crc32

5.2.1 bin2hex

Namespace: aloe.encoding
filter bin2hex : pipe<byte> -> pipe<string>

Converts a byte stream to a hexadecimal string representation.

5.2.2 crc32

Namespace: aloe.crypto.hash
filter crc32 : pipe<byte> -> pipe<uint32>

Computes the CRC32 of the entire input and outputs it as a single integer.

5.3 Base64 / URL

Namespace: aloe.encoding
filter base64encode : pipe<byte> -> pipe<string>
filter base64decode : pipe<string> -> pipe<byte>

Namespace: aloe.net.http
filter urlencode : pipe<string> -> pipe<string>
filter urldecode : pipe<string> -> pipe<string> 

fileBytes
    |: base64encode
    | stdout;

// ...

stdin
    |: base64decode
    | fileBytes;

5.4 Symmetric cryptography: Blowfish / AES

5.4.1 Blowfish: blowfishencrypt / blowfishdecrypt

Namespace: aloe.crypto.cipher
filter blowfishencrypt(ctx: BlowfishContext) : pipe<byte> -> pipe<byte>
filter blowfishdecrypt(ctx: BlowfishContext) : pipe<byte> -> pipe<byte>

Uses a BlowfishContext containing key, IV, mode, etc., to perform encryption and decryption.

5.4.2 AES: AESencrypt / AESdecrypt

Namespace: aloe.crypto.cipher
filter AESencrypt(ctx: AESContext) : pipe<byte> -> pipe<byte>
filter AESdecrypt(ctx: AESContext) : pipe<byte> -> pipe<byte>

Performs encryption/decryption using AES (e.g. CBC / GCM). Supported modes and padding details are defined by AESContext.

6. String Utilities

6.1 Case conversion: tolower / toupper

Namespace: aloe.text
filter tolower : pipe<string> -> pipe<string>
filter toupper : pipe<string> -> pipe<string> 

lines
    |: tolower
    | normalized;

6.2 Case styles: camelcase / pascalcase / snakecase

Namespace: aloe.text
filter camelcase : pipe<string> -> pipe<string>
filter pascalcase : pipe<string> -> pipe<string>
filter snakecase : pipe<string> -> pipe<string>

  • camelcase: "hello world" → "helloWorld"
  • pascalcase: "hello world" → "HelloWorld"
  • snakecase: "Hello World" → "hello_world"

6.3 Substring filters: startswith / contains / strstr

Namespace: aloe.text
filter startswith(prefix: string) : pipe<string> -> pipe<string>
filter contains(needle: string) : pipe<string> -> pipe<string>
filter strstr(needle: string) : pipe<string> -> pipe<string>

  • startswith: passes only lines that start with prefix.
  • contains: passes only lines that contain needle.
  • strstr: returns the substring from the first occurrence of needle to the end (similar to PHP’s strstr).
lines
    |: startswith: "[ERROR]"
    |: strstr: "detail="
    | stdout;

6.4 Trimming / Joining: ltrim / rtrim / trim / join

Namespace: aloe.text
filter ltrim : pipe<string> -> pipe<string>
filter rtrim : pipe<string> -> pipe<string>
filter trim : pipe<string> -> pipe<string>
filter join(sep: string) : pipe<string> -> pipe<string>

join concatenates all input strings using sep and outputs a single string. 

lines
    |: trim
    |: join: "\n"
    | text;

7. Archive / Compression / CSV

7.1 ZIP / TAR: zip / unzip / tar

Namespace: aloe.archive
filter zip : pipe<byte> -> pipe<byte>
filter unzip : pipe<byte> -> pipe<byte>
filter tar : pipe<byte> -> pipe<byte>

A simple model where the entire stream is treated as a single archive. Operations for multiple entries, filenames, etc. can be defined separately as higher-level APIs. 

stdin
    |: zip
    | stdout;

// ...

stdin
    |: unzip
    | stdout;

8. HTML / Sanitization

8.1 Escaping / Unescaping

Namespace: aloe.html
filter htmlspecialchars : pipe<string> -> pipe<string>
filter htmlspecialchars_decode : pipe<string> -> pipe<string>
filter html_entity_decode : pipe<string> -> pipe<string>
filter strip_tags : pipe<string> -> pipe<string>

  • htmlspecialchars: converts characters like <, >, &, " to HTML-safe entities.
  • htmlspecialchars_decode: inverse of the above.
  • html_entity_decode: decodes a wider range of HTML entities (e.g. &nbsp;).
  • strip_tags: removes HTML tags and leaves only text.
htmlLines
    |: strip_tags
    |: trim
    | plainText;

9. Parsing / Formatting

9.1 Query strings: parse_str

Namespace: aloe.net.http
filter parse_str : pipe<string> -> pipe<map<string, string>>

Parses query strings such as "foo=1&bar=2" and emits a map of key–value pairs.

9.2 Number formatting: number_format

Namespace: aloe.number
filter number_format(spec: string) : pipe<decimal> -> pipe<string>

Formats numbers as strings. The format string spec is implementation-dependent; for example, "#,##0.00" can be used.

9.3 Regex escaping: quote_meta

Namespace: aloe.text
filter quote_meta : pipe<string> -> pipe<string>

Escapes regular-expression meta characters so that the string can be safely used as a literal pattern. 

patterns
    |: quote_meta
    | safePatterns;

Filters not listed in this document can be freely added by users or external libraries. The names defined here are reserved as “Aloe standard filters” and are intended to serve as a common API across compatible implementations.
