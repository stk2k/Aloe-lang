この文書は、英語版仕様 https://stk2k.github.io/Aloe-lang/Documents/en/Aloe-lang-spec-en.html をベースにした日本語ドラフトです。
その上で、参照の無効状態を valid / invalid によって扱う「有効性モデル」を導入し、 null を仕様書から排除しています。
また 4.2 with-blocks に class / struct / enum を対象にした型ベース with（static メンバー／列挙子への短縮アクセス） の拡張を含みます。

0. Overview（概要）

Aloe は静的型付けのスクリプト言語であり、シンプルで直感的な構文を目指している。
スタックベースの仮想マシン（AloeVM）上で動作し、主に次のような目標を持つ。

  • 教育用途や小規模アプリケーションで読み書きしやすいこと
  • C# ランタイムおよび WebAssembly (WASM) 環境で動作すること
  • pipe と filter によって、I/O と並行処理を静的型付けのまま制御できること

特徴：

  • 静的型付け
    • var による型推論
    • let による明示的な型指定
  • 値型 / 参照型
    • struct ... 値型
    • class ... 参照型
  • GC ベースのメモリ管理（具体的なアルゴリズムは VM 実装依存）
  • 参照の無効状態（invalid）を言語レベルのモデルとして導入する
    • delete は「参照を無効化する命令」であり、無効化後の参照アクセスは例外となる。
    • invalid は代入や初期化に使えない（有効性チェック用途のみ）。
  • object を基底型とする参照型体系と、プリミティブの boxing / unboxing を導入する
    • プリミティブ値は object に代入できる（暗黙 boxing）。
    • unboxing は as 演算子と ifnot 句で行う（15章参照）。
  • pipe / filter によるデータフロースタイルの設計

1. Basic Syntax（基本文法）

1.1 Comments（コメント）

// single-line comment

/* multi-line
   comment */
  • // から行末までが行コメント。
  • /* ～ */ が複数行コメント。
  • 複数行コメントは、開始 /* から最初に現れる */ までを まるごとコメントとして扱う。 コメントの内部に追加の /* や */ が現れても、 それらはすべてコメントの一部として無視される（ネストしたコメントの構文解析は行わない）。
    したがって、コメント内の「コメント構文」が破綻していても、 いったん /* が始まってから最初の */ が来るまでは常にコメント扱いとなる。

1.2 Identifiers（識別子）

  • 使用可能文字：英字 (A-Z, a-z)、数字 (0-9)、アンダースコア _
  • 先頭文字は英字または _ でなければならない。
  • 大文字小文字は区別される。
  • キーワードと同じ綴りは使用できない。
  • _ は通常の識別子として使用できる（特別扱いしない）。
  • ユニコード文字も使用できる（空白や区切り記号などは除外）。

1.4 Literals（リテラル）

  • 整数リテラル：10, 0, -5, 0x1A, 0b1010 など
  • 浮動小数点リテラル：10.5, 0.0, -3.14
  • 10進リテラル：10.12345678901234567890:d など
  • 真偽値：true, false
  • 文字列："Hello, World!"
  • 文字：'A', 'z' など
  • 参照有効性：valid, invalid（有効性チェック専用。代入・初期化には使用できない）
  • 配列リテラル：[1, 2, 3]
  • リストリテラル：(1, 2, 3)
  • セットリテラル：{1, 2, 3}
  • マップリテラル：["key1": 10, "key2": 20]
  • enum リテラル：EnumType.Value1
  • struct リテラル：Point { x: 0, y: 1 }
  • オブジェクト生成：new ClassName()

1.5 Statement terminator（文の終端）

var x = 10;
x = x + 1;
print(x);
  • すべての文はセミコロン ; で終わる。
  • 改行による自動セミコロン挿入は行わない。

1.6 Whitespace（空白）

  • スペース、タブ、改行はトークン区切りとして扱われる。
  • 連続する空白は 1 つの区切りとみなされる。
  • インデントに意味はない（Python とは異なる）。

1.7 Case sensitivity（大文字小文字）

  • 識別子とキーワードは大文字小文字を区別する。
  • キーワードはすべて小文字（Result を除く）。

2. Type System（型システム）

2.1 Primitive types（プリミティブ型）

Aloe における「プリミティブ型」は、次の 8 種類とする。

  • bool / boolean - 真偽値。
  • int / integer - 符号付き整数。
  • float - 浮動小数点数。
  • decimal - 高精度 10 進数。
  • string - 文字列（内部的には Unicode）。
  • byte - 0-255 の符号なし整数。
  • char - 1 文字。
  • void - 戻り値なし（戻り値専用）。

valid / invalid は型ではなく、 参照の有効性を判定するための特殊定数（リテラル）である。
仕様書から null は排除し、参照の無効状態は invalid で表現する。

2.2 Composite / user-defined types（合成型 / ユーザー定義型）

  • 配列：T[]
  • リスト：T()
  • セット：T{}
  • マップ：map<T> または map<K/V>
  • struct
  • class
  • enum, bitfield enum
  • 非同期タスク：task
  • パイプ：pipe<T>（18章参照）
  • 結果型：Result<T>（18章参照）

2.3 Value types / reference types（値型 / 参照型）

  • 値型：
    • int, float, decimal, bool, byte, char
    • struct
    • enum, bitfield enum（内部表現は int32）
  • 参照型：
    • object（全参照型の基底型）
    • class
    • string（実装上は参照型）
    • task, pipe<T>, Result<T> など
    • プリミティブの box object（2.4 参照）

Struct restrictions and copying（struct の制約とコピー）

  • struct のフィールドにはプリミティブ型と他の struct のみを持てる。
  • class などの参照型フィールドは持てない。
  • struct の代入は常に値コピー。
  • 最大サイズなどは仕様で制限しない。

Class vs struct（class と struct）

  • class は struct をフィールドとして持てる。
  • class インスタンスを別変数に代入すると参照がコピーされる（オブジェクト本体は共有）。

2.4 object と box object（boxing / unboxing）

Aloe は object を参照型体系の基底型として導入する。
また、プリミティブ値を object として扱うために、対応する box object を提供する。

2.4.1 box object の命名規則

  • box object は大文字で始まる。
  • プリミティブ型に対応する英単語を用いる。
  • 別名を持つプリミティブについては、box object 名は次の正準名に固定する：
    • bool / boolean - Boolean
    • int / integer - Integer

2.4.2 box object 一覧

  • Boolean（bool を保持）
  • Integer（int を保持）
  • Float（float を保持）
  • Decimal（decimal を保持）
  • Byte（byte を保持）
  • Char（char を保持）

2.4.3 box object の公開インタフェース

  • box object の内部値は private フィールドであり、外部から直接参照できない。
  • unboxing は公開メソッド ToXxx() によって行う。
class Integer extends object {
    private field value: int;

    construct(v: int) {
        this.value = v;
    }

    public method ToInteger(): int {
        return this.value;
    }
}

2.4.4 暗黙 boxing（プリミティブ -> object）

  • 任意のプリミティブ値は object へ暗黙に変換できる（boxing）。
  • boxing は対応する box object を生成して格納する。
let o: object = 123;      // boxing: Integer
let p: object = true;     // boxing: Boolean

2.4.5 暗黙 unboxing（object -> プリミティブ）は存在しない

  • object からプリミティブへの暗黙変換は行わない。
  • unboxing は as 演算子と ifnot 句で行う（15章参照）。

3. Numbers & Conversions（数値と暗黙変換）

3.1 int and float

  • int と float を混在させた式の結果は float になる。
    • int + float -> float
    • int - float -> float
    • int * float -> float
    • int / float -> float

3.2 decimal

  • decimal は高精度な 10 進数。
  • int / float と混在する式は、必要に応じて decimal に昇格しうる。
  • オーバーフロー / アンダーフロー / 精度問題は OverflowException として扱う。

3.3 enum and int

  • Aloe は enum と int の暗黙 / 明示変換はいずれも禁止する。
  • enum と数値型の混在式はコンパイルエラー（bitfield enum のビット演算を除く）。
  • 内部表現としては enum / bitfield enum はすべて int32。
  • as 演算子はプリミティブ unboxing 用途に限る（15章参照）。

3.4 byte and char

  • byte：
    • 範囲は 0-255。
    • byte 同士の演算は許されるが、範囲外はエラーになり得る。
    • byte と int の混在式は int に昇格。
  • char：
    • char の算術演算は不可。
    • 比較演算（==, <, > 等）は char 同士のみ。
    • 数値型との暗黙変換はない。

3.5 NaN / Infinity

  • 数値オーバーフロー/アンダーフローは NaN/Infinity ではなく例外で扱う。
  • ZeroDivisionException は 0 / 0.0 で常に投げる。

4. Blocks & Scope（ブロックとスコープ）

ブロックは { と } で区切られ、各ブロックが新しいスコープを導入する。

  • main ブロック
  • クラス定義ブロック
  • メソッド定義ブロック
  • 任意の { ... } ブロック

4.1 Static (lexical) scope and shadowing（静的スコープとシャドウイング）

class Sample {
    field value: int = 1;
    method foo(): void {
        var value = 2;        // フィールドをシャドウ
        print(value);         // 2
        print(this.value);    // 1
    }
}
  • Aloe は静的（レキシカル）スコープ。
  • 内側ブロックで同名変数を宣言すると、外側の変数をシャドウする。
  • フィールド名とローカル変数名がぶつかった場合、ローカル変数が優先される。フィールドは this. で参照する。

4.2 with-blocks（メンバー省略記法）

Aloe には、同一オブジェクトのメンバーアクセスを簡潔に書くための with ブロックがある。 

with (user) {
    .name = "Alice";
    .age = .age + 1;
    print(.name);
}
  • with (expr) { ... } ブロック内では：
    • .Member は (expr).Member の省略形。
    • expr はブロックの先頭で 1 度だけ評価され、内部で保持される（再評価しない）。
  • _ は特別扱いしない。 with 対象を表す暗黙変数は存在しない。
  • ネストした with ブロック：
    • 内側の with が . の解決対象となる。
    • 外側オブジェクトへは通常の変数名でアクセスできる。
  • C# の using スタイルのように、同じブロックに複数の with を並べられる：
with (user), (profile) {
    .name = "Alice";  // resolved against user, then profile
}
  • 解決順序は 左から右 とする。
  • 左から探索して最初に見つかったメンバーに解決する。
  • ただし同名メンバーが複数ターゲットに存在し、解決が曖昧になる場合は コンパイルエラー。

with は構文糖衣であり、意味的にはすべて明示的な obj.member 呼び出しに書き換えた上で 型チェック・コード生成を行うことができる。

4.2.1 Type-based with（class / struct / enum の static / 列挙子ショートカット）

通常の with (expr) に加えて、型名 を対象にした with を利用できる。 

with (TypeName) {
    .SomeStaticMethod();
    .SomeStaticField = 1;
}

ここで TypeName は次のいずれか：

  • class 型
  • struct 型
  • enum / bitfield enum

この場合、with は「型コンテキスト with」として解釈される：

  • () 内の式は実行時に値として評価されない。
    • コンパイラは TypeName を型として解決し、インスタンス生成などは行わない。
  • ブロック内では：
    • .Member は TypeName.Member の省略形。
    • 型ベース with 内でインスタンスメンバーにアクセスすることはコンパイルエラー。

型ベース with も複数ターゲット構文に混在させられる。解決順序は左から右であり、曖昧ならコンパイルエラーとする。

5. Variables & Constants（変数と定数）

5.1 var（型推論）

var i = 10;
var f = 10.5;
var s = "Hello";
var c = 'A';
var b = 255;
var arr = [1, 2, 3];
var lst = (1, 2, 3);
var st  = {1, 2, 3};
var mp  = ["key": 10];
var fb  = new FizzBuzz();
  • 変数の型は右辺の式から推論される。
  • var x = 1; は常に int。
  • var x = 1.0; は float として扱われる。
  • var x = invalid; はコンパイルエラー（invalid は代入・初期化に使用できない）。
  • 右辺の式の型が object で固定できる場合、推論結果の型も object になる。
function SomeFunc(): object {
    return 123; // boxing: Integer
}

function main(args: string[]): int {
    var x = 1;            // int
    var o = SomeFunc();   // object
    return 0;
}

5.2 let（明示的な型）

let i: int = 10;
let f: float = 10.5;
let d: decimal = 10.12345678901234567890:d;
let s: string = "Hello, Aloe!";
let c: char = 'A';
let b: byte = 255;

let arr: int[] = [1, 2, 3];
let lst: int() = (1, 2, 3);
let st : int{} = {1, 2, 3};
let mp : map<string/int> = ["key": 10];

let o: object = 123; // boxing: Integer
  • let x: Foo = invalid; はコンパイルエラー（invalid は代入・初期化に使用できない）。

5.2.1 未初期化参照（uninitialized）

参照型変数は、初期化を省略した宣言を許可する。 この場合、その参照は invalid 状態として扱われる（未初期化と invalid は区別しない）。 

var obj: Foo;        // OK（参照型の未初期化宣言）
