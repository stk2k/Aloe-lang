0. Overview（概要）

Aloe は静的型付けのスクリプト言語であり、シンプルで直感的な構文を目指している。
スタックベースの仮想マシン（AloeVM）上で動作し、主に次のような目標を持つ。

  • 教育用途や小規模アプリケーションで読み書きしやすいこと
  • C# ランタイムおよび WebAssembly (WASM) 環境で動作すること
  • pipe と filter によって、I/O と並行処理を静的型付けのまま制御できること

特徴：

  • 静的型付け
    • var による型推論
    • let による明示的な型指定
  • 値型 / 参照型
    • struct ... 値型
    • class / object ... 参照型
  • GC ベースのメモリ管理（具体的なアルゴリズムは VM 実装依存）
  • 参照の無効状態（invalid）を言語レベルのモデルとして導入する
    • delete は「参照を無効化する命令」であり、無効化後の参照アクセスは例外となる。
    • invalid は代入や初期化に使えない（有効性チェック用途のみ）。
  • object による汎用参照と box object による boxing / unboxing
  • pipe / filter によるデータフロースタイルの設計

1. Basic Syntax（基本文法）

1.1 Comments（コメント）

// single-line comment

/* multi-line
   comment */
  • // から行末までが行コメント。
  • /* ～ */ が複数行コメント。
  • 複数行コメントは、開始 /* から最初に現れる */ までを まるごとコメントとして扱う。 コメントの内部に追加の /* や */ が現れても、 それらはすべてコメントの一部として無視される（ネストしたコメントの構文解析は行わない）。
    したがって、コメント内の「コメント構文」が破綻していても、 いったん /* が始まってから最初の */ が来るまでは常にコメント扱いとなる。

1.2 Identifiers（識別子）

  • 使用可能文字：英字 (A-Z, a-z)、数字 (0-9)、アンダースコア _
  • 先頭文字は英字または _ でなければならない。
  • 大文字小文字は区別される。
  • キーワードと同じ綴りは使用できない。
  • _ は通常の識別子として使用できる（特別扱いしない）。
  • ユニコード文字も使用できる（空白や区切り記号などは除外）。

1.4 Literals（リテラル）

  • 整数リテラル：10, 0, -5, 0x1A, 0b1010 など
  • 浮動小数点リテラル：10.5, 0.0, -3.14
  • 10進リテラル：10.12345678901234567890:d など
  • 真偽値：true, false
  • 文字列："Hello, World!"
  • 文字：'A', 'z' など
  • 参照有効性：valid, invalid（有効性チェック専用。代入・初期化には使用できない）
  • 配列リテラル：[1, 2, 3]
  • リストリテラル：(1, 2, 3)
  • セットリテラル：{1, 2, 3}
  • マップリテラル：["key1": 10, "key2": 20]
  • enum リテラル：EnumType.Value1
  • struct リテラル：Point { x: 0, y: 1 }
  • オブジェクト生成：new ClassName()

1.5 Statement terminator（文の終端）

var x = 10;
x = x + 1;
print(x);
  • すべての文はセミコロン ; で終わる。
  • 改行による自動セミコロン挿入は行わない。

1.6 Whitespace（空白）

  • スペース、タブ、改行はトークン区切りとして扱われる。
  • 連続する空白は 1 つの区切りとみなされる。
  • インデントに意味はない（Python とは異なる）。

1.7 Case sensitivity（大文字小文字）

  • 識別子とキーワードは大文字小文字を区別する。
  • キーワードはすべて小文字（Result を除く）。

2. Type System（型システム）

2.1 Primitive types（プリミティブ型）

Aloe における「プリミティブ型」は、次の 8 種類とする。

  • bool / boolean - 真偽値。
  • int / integer - 符号付き整数。
  • float - 浮動小数点数。
  • decimal - 高精度 10 進数。
  • string - 文字列（内部的には Unicode）。
  • byte - 0-255 の符号なし整数。
  • char - 1 文字。
  • void - 戻り値なし（戻り値専用）。

valid / invalid は型ではなく、 参照の有効性を判定するための特殊定数（リテラル）である。
仕様書から null は排除し、参照の無効状態は invalid で表現する。

2.2 Composite / user-defined types（合成型 / ユーザー定義型）

  • 配列：T[]
  • リスト：T()
  • セット：T{}
  • マップ：map<T> または map<K/V>
  • struct
  • class
  • enum, bitfield enum
  • 非同期タスク：task
  • 汎用参照：object
  • パイプ：pipe<T>（18章参照）
  • 結果型：Result<T>（18章参照）

2.3 Value types / reference types（値型 / 参照型）

  • 値型：
    • int, float, decimal, bool, byte, char
    • struct
    • enum, bitfield enum（内部表現は int32）
  • 参照型：
    • class
    • string（実装上は参照型）
    • object
    • task, pipe<T>, Result<T> など

object type（object 型）

  • object は、void を除く任意の値を保持できる特別な参照型である。
  • 任意の参照型の値は object へ暗黙に代入できる（参照コピー）。
  • プリミティブ型の値を object に代入する場合、boxing により対応する box object に変換される（2.4 参照）。
  • object から具体型への暗黙変換はない（型変換は as または暗黙 unbox キャストにより行う）。
  • if (x is T) { ... } のブロック内では、x を T として扱ってよい（フロー依存の型絞り込み）。

Struct restrictions and copying（struct の制約とコピー）

  • struct のフィールドにはプリミティブ型と他の struct のみを持てる。
  • class などの参照型フィールドは持てない。
  • struct の代入は常に値コピー。
  • 最大サイズなどは仕様で制限しない。

Class vs struct（class と struct）

  • class は struct をフィールドとして持てる。
  • class インスタンスを別変数に代入すると参照がコピーされる（オブジェクト本体は共有）。

2.4 boxing / unboxing（box object と ToXxx）

Aloe は object を介してプリミティブ値を扱うために、プリミティブごとに対応する box object（クラス）を定義する。 box object は大文字で始まる名前を持ち、object から派生する。

  • bool / boolean - Boolean
  • int / integer - Integer
  • float - Float
  • decimal - Decimal
  • string - String
  • byte - Byte
  • char - Char
  • box object は内部に対応する値を保持するが、その値を直接参照する公開プロパティは持たない（value は非公開）。
  • unboxing は、対応する ToXxx() メソッドにより行う。
  • ToXxx() の正準名（canonical）は次の通り：
    • ToBool(), ToInt(), ToFloat(), ToDecimal(), ToString(), ToByte(), ToChar()
  • boxing：プリミティブ値を object に代入する場合、対応する box object のインスタンスを生成して格納する。
  • unboxing：object からプリミティブを取り出す場合、as 演算子または暗黙 unbox キャスト、unbox 演算子を用いる。

3. Numbers & Conversions（数値と暗黙変換）

3.1 int and float

  • int と float を混在させた式の結果は float になる。
    • int + float -> float
    • int - float -> float
    • int * float -> float
    • int / float -> float

3.2 decimal

  • decimal は高精度な 10 進数。
  • int / float と混在する式は、必要に応じて decimal に昇格しうる。
  • オーバーフロー / アンダーフロー / 精度問題は OverflowException として扱う。

3.3 enum and int

  • Aloe には括弧キャスト構文（例：(T)x）はない。
  • 型変換は as 演算子により行う（15.1 参照）。
  • enum と int の暗黙 / 明示変換はいずれも禁止。
  • enum と数値型の混在式はコンパイルエラー（bitfield enum のビット演算を除く）。
  • 内部表現としては enum / bitfield enum はすべて int32。

3.4 byte and char

  • byte：
    • 範囲は 0-255。
    • byte 同士の演算は許されるが、範囲外はエラーになり得る。
    • byte と int の混在式は int に昇格。
  • char：
    • char の算術演算は不可。
    • 比較演算（==, <, > 等）は char 同士のみ。
    • 数値型との暗黙変換はない。

3.5 NaN / Infinity

  • 数値オーバーフロー/アンダーフローは NaN/Infinity ではなく例外で扱う。
  • ZeroDivisionException は 0 / 0.0 で常に投げる。

4. Blocks & Scope（ブロックとスコープ）

ブロックは { と } で区切られ、各ブロックが新しいスコープを導入する。

  • main ブロック
  • クラス定義ブロック
  • メソッド定義ブロック
  • 任意の { ... } ブロック

4.1 Static (lexical) scope and shadowing（静的スコープとシャドウイング）

class Sample {
    field value: int = 1;
    method foo(): void {
        var value = 2;        // フィールドをシャドウ
        print(value);         // 2
        print(this.value);    // 1
    }
}
  • Aloe は静的（レキシカル）スコープ。
  • 内側ブロックで同名変数を宣言すると、外側の変数をシャドウする。
  • フィールド名とローカル変数名がぶつかった場合、ローカル変数が優先される。フィールドは this. で参照する。

4.2 with-blocks（メンバー省略記法）

Aloe には、同一オブジェクトのメンバーアクセスを簡潔に書くための with ブロックがある。 

with (user) {
    .name = "Alice";
    .age = .age + 1;
    print(.name);
}
  • with (expr) { ... } ブロック内では：
    • .Member は (expr).Member の省略形。
    • expr はブロックの先頭で 1 度だけ評価され、内部で保持される（再評価しない）。
  • _ は特別扱いしない。 with 対象を表す暗黙変数は存在しない。
  • ネストした with ブロック：
    • 内側の with が . の解決対象となる。
    • 外側オブジェクトへは通常の変数名でアクセスできる。
  • C# の using スタイルのように、同じブロックに複数の with を並べられる：
with (user), (profile) {
    .name = "Alice";  // resolved against user, then profile
}
  • 解決順序は 左から右 とする。
  • 左から探索して最初に見つかったメンバーに解決する。
  • ただし同名メンバーが複数ターゲットに存在し、解決が曖昧になる場合は コンパイルエラー。

with は構文糖衣であり、意味的にはすべて明示的な obj.member 呼び出しに書き換えた上で 型チェック・コード生成を行うことができる。

4.2.1 Type-based with（class / struct / enum の static / 列挙子ショートカット）

通常の with (expr) に加えて、型名 を対象にした with を利用できる。 

with (TypeName) {
    .SomeStaticMethod();
    .SomeStaticField = 1;
}

ここで TypeName は次のいずれか：

  • class 型
  • struct 型
  • enum / bitfield enum

この場合、with は「型コンテキスト with」として解釈される：

  • () 内の式は実行時に値として評価されない。
    • コンパイラは TypeName を型として解決し、インスタンス生成などは行わない。
  • ブロック内では：
    • .Member は TypeName.Member の省略形。
    • 型ベース with 内でインスタンスメンバーにアクセスすることはコンパイルエラー。

型ベース with も複数ターゲット構文に混在させられる。解決順序は左から右であり、曖昧ならコンパイルエラーとする。

5. Variables & Constants（変数と定数）

5.1 var（型推論）

var i = 10;
var f = 10.5;
var s = "Hello";
var c = 'A';
var b = 255;
var arr = [1, 2, 3];
var lst = (1, 2, 3);
var st  = {1, 2, 3};
var mp  = ["key": 10];
var fb  = new FizzBuzz();
  • 変数の型は右辺の式から推論される。
  • var x = 1; は常に int。
  • var x = 1.0; は float として扱われる。
  • var x = invalid; はコンパイルエラー（invalid は代入・初期化に使用できない）。
  • object 型の式を右辺に置いた場合、推論結果の型も object になる。
function SomeFunc(): object {
    return 123;   // boxing to Integer
}

function main(args: string[]): int {
    var x = 1;            // int
    var a = 123;          // int
    var v = SomeFunc();   // object
    return 0;
}

5.2 let（明示的な型）

let i: int = 10;
let f: float = 10.5;
let d: decimal = 10.12345678901234567890:d;
let s: string = "Hello, Aloe!";
let c: char = 'A';
let b: byte = 255;

let arr: int[] = [1, 2, 3];
let lst: int() = (1, 2, 3);
let st : int{} = {1, 2, 3};
let mp : map<string/int> = ["key": 10];
  • let x: Foo = invalid; はコンパイルエラー（invalid は代入・初期化に使用できない）。

5.2.1 未初期化参照（uninitialized）

参照型変数は、初期化を省略した宣言を許可する。 この場合、その参照は invalid 状態として扱われる（未初期化と invalid は区別しない）。 

var obj: Foo;        // OK（参照型の未初期化宣言）
obj.Value;           // 実行時例外（InvalidReferenceException）
  • 初期化を省略した参照型変数は、実行時に invalid とみなされる。
  • 無効参照（invalid）へのメンバーアクセスは InvalidReferenceException を投げる。

5.2.2 暗黙 unbox キャスト（消費しない）

「期待型（expected type）がプリミティブ型 P に固定される文脈」では、 式 E に対して 暗黙 unbox キャストを許可する。 暗黙 unbox キャストは 参照を消費しない（参照変数を invalid 化しない）。

暗黙 unbox キャストが許可される文脈の例：

  • let x:P = E;（初期化）
  • x = E;（代入。x の型が P）
  • return E;（戻り値型が P）
  • f(E)（仮引数型が P）

暗黙 unbox キャストは、意味的に E as P と等価であり、失敗時は InvalidCastException を投げる。 ifnot による代替値指定は、明示 as のみで行う（15.1 参照）。

成功条件（変換は行わない）：

  • E の実行時値がプリミティブ P である。
  • または、E の実行時値が対応する box object（例：Integer）であり、ToXxx() により P を取り出せる。

禁止：

  • 数値型間の変換（例：Integer を byte に暗黙変換しない）。
  • 文字列パース等の変換（例："123" を int に暗黙変換しない）。
let o: object = 123;      // boxing: Integer
let i: int = o;           // OK（暗黙 unbox キャスト） == (o as int)

let s2: string = o;       // 失敗（変換しない） throws InvalidCastException

5.3 Constants（const）

const MaxCount: int = 100;
const Pi: float = 3.14159;
const AppName: string = "Aloe VM";
  • const 変数は再代入不可。
  • const には型注釈が必須。

6. Fields（フィールド宣言）

class / struct のフィールドは field キーワードで宣言する。

class Sample {
    field count: int = 0;
    field name: string = "default";

    readonly field id: int;

    construct(id: int) {
        this.id = id;
    }
    method change(): void {
        this.count = this.count + 1;
    }
}
  • field ... 通常のフィールド。
  • readonly field ... コンストラクタ内のみ代入可、それ以降は読み取り専用。
  • struct / class の両方で同じ意味。

7. Class / Struct / Interface / Trait

7.1 class

class FizzBuzz {
    field _count: int = 0;
    field name: string = "FizzBuzz";

    construct() {
    }

    method run(): void {
        var i = 1;
        while (i <= 100) {
            i = i + 1;
        }
    }
}
  • class は参照型。
  • sealed class は extends による継承不可。
  • 単一継承のみサポート。

7.2 struct

struct Point {
    field x: int;
    field y: int;

    construct(x: int, y: int) {
        this.x = x;
        this.y = y;
    }
}
  • struct は値型。
  • フィールドはプリミティブ型か他の struct に限定（参照型フィールドは禁止）。
  • struct のコピーは常に値コピー。
  • struct の継承はサポートしない（sealed struct は「拡張禁止」マーカー）。

Struct layout annotations: @StructLayout / @FieldOffset

8. delete & invalid

8.1 参照の有効性モデル（valid / invalid）

Aloe では、参照が「有効（valid）」か「無効（invalid）」かという状態をモデル化する。
invalid は 代入や初期化に使用できないが、参照の有効性チェックには使用できる。 

var obj = new FizzBuzz();
delete obj;           // obj becomes invalid
if (obj is invalid) {
    print("invalid");
}
  • delete x; は「参照変数 x を無効化する命令」である。
  • delete は例外を投げてはならない。 既に invalid の場合も無効果である。
  • 無効化後、x を通じた参照先のメンバーアクセスは InvalidReferenceException を引き起こす。
  • 同一オブジェクトを参照している別の参照変数には干渉しない（参照の無効化は変数単位）。
  • オブジェクトの実際の解放（回収）は実行系が決定する（GC）。
  • 未初期化参照と invalid は区別しない。 初期化を省略した参照型変数は invalid とみなされる。
  • invalid は比較・判定用であり、x = invalid や let x: Foo = invalid はコンパイルエラー。

8.2 invalid 参照アクセス

var obj: Foo;     // invalid
obj.Value;        // throws InvalidReferenceException
  • invalid 参照へのメンバーアクセスは必ず InvalidReferenceException。
  • （実装が静的に invalid を確定できても）原則として挙動は同じとし、最適化は実装依存とする。

15. Operators（演算子）

15.1 as 演算子と ifnot（型変換と代替値）

Aloe は括弧キャストを持たず、型変換には as 演算子を用いる。 as は失敗時に InvalidCastException を投げる。 また ifnot 節により、失敗時の代替値または throw を指定できる。 

let o: object = 123;                 // boxing: Integer

let i1: int = o as int;              // OK（unboxing）失敗なら InvalidCastException
let i2: int = o as int ifnot -1;     // 失敗なら -1
let i3: int = o as int ifnot throw;  // 失敗なら InvalidCastException（省略時と同じ）
  • E as T の T が参照型の場合：実行時型チェックによりキャストを行う。
  • E as P の P がプリミティブの場合：
    • E がプリミティブ P ならそのまま成功。
    • E が対応する box object（例：Integer）なら ToXxx() により unboxing して成功。
    • それ以外は失敗（InvalidCastException）。
  • ifnot 節が省略された場合は ifnot throw と同義。
  • ifnot throw は InvalidCastException を投げる。
  • as は変換を行わない（数値型間変換や文字列パース等は行わない）。

15.3 unbox 演算子（消費する unboxing）

unbox は、参照変数からプリミティブ値を取り出し、成功時に参照変数を invalid 化する演算子である。 これは「消費する unboxing（move 的 unboxing）」である。 

let o: object = 123;         // boxing: Integer
let i: int = unbox o;        // i = 123, o becomes invalid
  • unbox のオペランドは「代入可能な参照変数（lvalue）」でなければならない。
  • unbox x は、期待型がプリミティブ P に固定される文脈でのみ使用できる。
  • 評価規則（x は 1 回だけ評価される）：
    • x が invalid の場合：InvalidReferenceException を投げる（x は変化しない）。
    • unboxing が失敗する場合：InvalidCastException を投げる（x は変化しない）。
    • unboxing が成功する場合：値を返し、その後に delete x; を実行したのと同じく x を invalid 化する。
  • unbox は暗黙 unbox キャストと異なり、成功時に参照変数を消費する。
